Build & quality contract

Scope: reproducible local and CI verification for reader/. Versions live in reader/gradle/libs.versions.toml. Baseline: JDK 21, checked-in Gradle 9.1.0 wrapper, Android SDK 36, Xcode 26.4 or compatible 26 patch; Kotlin 2.4.10, Compose 1.11.1, AGP 9.0.1. Never use machine Gradle; wrapper JAR/scripts/properties/checksum are inputs.

Fresh checkout

  1. Install JDK 21 and Android Platform/Build Tools 36.
  2. On macOS install/select Xcode 26.
  3. In reader/, run ./gradlew --version.
  4. Set ANDROID_HOME or untracked local.properties with absolute sdk.dir.
  5. Run the gates below; first use downloads artifacts/Kotlin Native.

Unsigned iOS app

xcodebuild \
  -project iosApp/iosApp.xcodeproj \
  -scheme iosApp \
  -configuration Debug \
  -destination 'generic/platform=iOS Simulator' \
  -derivedDataPath build/ios-derived-data \
  CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO build

This compiles Swift, invokes :shared:embedAndSignAppleFrameworkForXcode, and links a simulator app without booting a device or requiring a team.

Invariants & limits

Android

Release keeps minification, resource shrinking, optimized defaults and project rules. Mapping check prevents silent R8 removal. App host uses com.android.application; shared uses AGP 9 KMP library.

iOS

Both device and arm64 simulator targets exist. Generic simulator build disables signing. First Native run may take minutes. Xcode 26.6 warns that bundled ICU targets Simulator 18.5 while the existing app target is 18.2; build succeeds, but 18.2 runtime needs release verification or deliberate target alignment.

Evidence boundary

Compilation does not prove VoiceOver, motion, transparency or physical-device behaviour. Xcode version is printed; update compatibility evidence before toolchain drift.
